# UCII

## ユニバーサル暗号化アイデンティティ基盤

**アイデンティティこそがプリミティブである。認証は単なるアプリケーションの一つにすぎない。**

ポスト量子暗号によって保護され、x402 経済認証によって強制管理される、プロダクション対応の暗号化アイデンティティ基盤。

---

## 概要

UCII（Universal Cryptographic Identity Infrastructure：ユニバーサル暗号化アイデンティティ基盤）は、次世代インターネットのために設計されたプロダクションレベルのアイデンティティ基盤です。

認証をシステムの基盤として扱うのではなく、UCII は **アイデンティティ（Identity）** を永続的な暗号学的プリミティブとして扱います。認証、認可、認証情報、決済、そしてアプリケーション固有の信頼は、アイデンティティを定義するものではなく、その上に構築されるサービスになります。

すべての保護された操作は最新のポスト量子暗号によって保護され、x402 決済プロトコルによって経済的に強制管理されます。

UCII は、人間、AI エージェント、自律システム、組織、サービス、接続デバイスのための安全な基盤を提供することを目的としています。

---

## なぜ UCII なのか？

従来のアイデンティティシステムは、ユーザー名、パスワード、OAuth プロバイダー、認証セッションを中心に構築されています。

UCII は異なるアプローチを採用します。

アイデンティティは、どのように認証されるかに依存せず存在すべきです。

パスワードは期限切れになります。

OAuth プロバイダーは変化します。

証明書はローテーションされます。

認証方式は進化します。

アイデンティティは永続的であるべきです。

UCII は暗号学的アイデンティティをルートオブジェクトとして確立し、基盤となるアイデンティティを置き換えることなく、認証方式、認証情報、認可メカニズムが進化できるようにします。

## アーキテクチャ

```text
                アプリケーション
                      │
                  認証
                      │
                  認可
                      │
              認証情報
                      │
          ユニバーサルアイデンティティ
                      │
          ポスト量子暗号
```

アイデンティティは永続します。

その他すべてはその上に構築されます。

---

## 経済認証（x402）

UCII は x402 プロトコルを統合し、インフラストラクチャサービスにネイティブな経済認証機能を提供します。

各保護対象エンドポイントは必要な支払い額を公開します。

クライアントは支払い証明を送信します。

UCII は決済を検証します。

決済レシートは永続的に記録されます。

リプレイ試行は拒否されます。

その後にのみ、保護された操作が実行されます。

経済認証は外部課金システムではなく、インフラストラクチャ自体の一部になります。

## パブリック API

### アイデンティティ作成

```text
POST /v1/identity
```

新しいユニバーサル暗号化アイデンティティを作成します。

**価格：0.50 USDC**

---

### 認証情報登録

```text
POST /v1/credentials/register
```

暗号学的認証情報を登録します。

**価格：0.75 USDC**

---

### 認証情報検証

```text
POST /v1/credentials/verify
```

登録済み認証情報を検証します。

**価格：0.10 USDC**

---

追加エンドポイントは OpenAPI 仕様を通じてドキュメント化されています。

## ポスト量子セキュリティとの整合性

UCII は ML-DSA および ML-KEM を含む NIST 標準化アルゴリズムを使用した、最新のポスト量子暗号を基盤として構築されています。

このプロジェクトは、量子耐性インフラストラクチャへの業界全体の移行に対応しており、組織がポスト量子移行期限より十分前に、プロダクション対応の暗号化アイデンティティシステムを導入できるよう支援することを目的としています。

## プロジェクトビジョン

UCII はアプリケーションではなく、インフラストラクチャを構築しています。

アイデンティティは普遍的であるべきです。

暗号技術は量子安全であるべきです。

認可は経済的に強制可能であるべきです。

開発者はアプリケーションごとにアイデンティティを再構築するのではなく、アイデンティティの上にアプリケーションを構築できるべきです。

UCII は、その基盤を提供するために存在します。
