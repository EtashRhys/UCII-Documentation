# UCII Identity Model

## Cryptographic Identity

UCII treats identity as the foundational object.

Authentication, credentials, and authorization are layered on top of identity.

---

## Identity Classes

### HUMAN

Human-controlled cryptographic identity.

### AI_AGENT

Autonomous software agent identity.

### ROBOT

Physical autonomous system identity.

### DEVICE

Connected hardware identity.

### SERVICE

Application or infrastructure service identity.

### ORGANIZATION

Institutional or organizational identity.

---

## Identity Lifecycle

A UCII identity can:

- be created
- receive credentials
- verify credentials
- rotate credentials
- revoke credentials

Identity remains independent from any single authentication mechanism.
